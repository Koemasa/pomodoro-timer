## 1. 定義

ガイドライン通則編 第2章（法第2条・第16条関係）より

### 個人情報（法第2条第1項）
生存する個人に関する情報であって、次のいずれかに該当するもの：
- 氏名、生年月日その他の記述等により**特定の個人を識別できるもの**
- **個人識別符号**が含まれるもの

**開発上の注意**：メールアドレス・IPアドレス単体でも、他の情報と組み合わせて個人を識別できれば「個人情報」に該当しうる。

### 個人識別符号（法第2条第2項）
以下が該当する（政令で定める）：
- 顔・虹彩・声紋・歩行の態様・手指の静脈・指紋など**身体的特徴を変換した符号**
- 旅券番号、基礎年金番号、免許証番号、住民票コード、マイナンバー、健康保険証の被保険者番号など**公的な番号**
- サービスの利用者・購入者・発行を受ける者ごとに異なるように割り当てられた**ID番号**（ユーザーIDなど）

### 要配慮個人情報（法第2条第3項）
取得に**本人の同意が必須**（オプトアウト不可）：

| 種別 | 具体例 |
|------|--------|
| 人種・民族 | 国籍、民族的出身 |
| 信条 | 政治的意見、宗教的信念 |
| 社会的身分 | 被差別部落出身等 |
| 病歴 | 既往症、投薬歴 |
| 犯罪の経歴 | 前科・前歴 |
| 犯罪被害 | 被害事実 |
| 障害 | 身体・知的・精神・発達障害等 |
| 健康診断結果 | 検査値、診断結果 |
| 性生活・性的指向・性自認 | （令和3年改正で追加） |

### 個人データ（法第16条第3項）
**個人情報データベース等**（体系的に整理された集合体）を構成する個人情報。  
→ DBやスプレッドシートに保存した時点で「個人データ」となり、より厳格な義務が発生する。

### 保有個人データ（法第16条第4項）
個人情報取扱事業者が開示・訂正・消去等を行う権限を有する個人データ。  
→ ユーザーに**開示請求権・訂正請求権・削除請求権**が発生する。

### 個人関連情報（法第2条第7項）
生存する個人に関する情報であって、個人情報・仮名加工情報・匿名加工情報のいずれにも該当しないもの。  
例：Cookie情報、位置情報、閲覧履歴、購買履歴（個人と紐付いていない状態のもの）

**注意**：第三者が個人関連情報を受け取り、それを個人データと照合して個人を識別する場合は「個人関連情報の第三者提供」の制限（法第31条）が適用される。

### 仮名加工情報（法第2条第5項）
他の情報と照合しない限り特定の個人を識別できないように個人情報を加工したもの。  
→ 内部分析・統計用途のみ可。第三者提供は原則不可（一部例外あり）。

### 匿名加工情報（法第2条第6項）
特定の個人を識別できず、かつ個人情報を復元できないように加工したもの。  
→ 本人同意なしで第三者提供可。ただし加工基準を遵守すること。

## 2. 個人情報取扱事業者の義務

ガイドライン通則編 第3章（法第17条〜第46条関係）より

### 利用目的の特定・制限（法第17条・第18条）

- 利用目的を**できる限り特定**すること
- 特定した目的の範囲を**超えて利用してはならない**
- 目的を変更する場合は変更前の目的と**関連性が認められる範囲**に限る

**NG例と OK例**：
```
× 「個人情報は適切に管理します」
× 「サービス向上のために利用します」
○ 「ご入力いただいた氏名・住所は、商品の発送のみに利用します」
○ 「メールアドレスは、お申込み確認メールおよびサービスに関する重要なお知らせの送信に利用します」
```

利用目的による制限の**例外**（法第18条第3項）：
- 法令に基づく場合
- 人の生命・身体・財産保護のため必要で本人同意の取得が困難な場合
- 公衆衛生・児童の健全育成のため必要で本人同意の取得が困難な場合
- 国の機関等への協力のため必要で同意取得が当該業務の遂行に支障を及ぼすおそれがある場合

### 不適正利用の禁止（法第19条）
違法または不当な行為を助長・誘発するおそれがある方法で個人情報を利用してはならない。

### 適正な取得（法第20条）
- **偽り**その他不正の手段により個人情報を取得してはならない
- 要配慮個人情報の取得には**本人の同意**が必要（例外あり）

要配慮個人情報の取得が不要な例外：
- 法令に基づく場合
- 人の生命・身体・財産保護のため必要で本人同意取得が困難な場合
- 本人が自ら公開している場合（SNS投稿等）
- 報道・研究・学術目的の場合（一定条件下）

### 利用目的の通知・公表（法第21条）
個人情報を取得した場合、**あらかじめ公表**しているか、**速やかに通知または公表**すること。

書面等で直接取得する場合（フォーム等）は取得時に**明示**すること。

通知等が不要な例外：
- 利用目的を本人に通知・公表することで本人または第三者の生命・身体・財産等に危害が生ずるおそれがある場合
- 取得の状況から見て利用目的が明らかな場合

## 3. 安全管理措置

ガイドライン通則編 別添「講ずべき安全管理措置の内容」より（法第23条関係）

### 基本方針の策定
- 関係法令・ガイドラインの遵守
- 質問・苦情処理の窓口の設置

### 個人データの取扱いに係る規律の整備
- 取得・入力・利用・出力・提供・保存・削除・廃棄の各段階でのルール策定

### 組織的安全管理措置
- 個人データの**安全管理に関する責任者**の設置
- 個人データを取り扱う**従業者の明確化**
- 取扱状況の確認手段の整備
- 漏えい等事案に対応する**体制の整備**
- 個人データの取扱状況の**定期的な点検・監査**

### 人的安全管理措置
- 個人データの取扱いに関する**従業者への教育・研修**
- 退職後も含めた**秘密保持義務**の周知

### 物理的安全管理措置
- 個人データを取り扱う区域の**管理（入退室管理等）**
- 機器・電子媒体等の**盗難・紛失対策**
- 電子媒体等を持ち運ぶ際の**漏えい防止措置**（暗号化等）
- 個人データの**削除・廃棄方法**の適正化（専用ソフトウェアによる上書き、物理的破壊等）

### 技術的安全管理措置
- 個人データを取り扱う**アクセス者の識別と認証**
- 個人データへの**アクセス権限の管理**（最小権限の原則）
- 個人データに対する**不正なアクセス等の防御措置**
  - ファイアウォール
  - 不正侵入検知システム（IDS/IPS）
  - ウイルス対策
- 個人データの**暗号化・パスワード設定等**
- ログの取得・保存

### 技術的安全管理の開発実装要点
```
必須：
- 通信の HTTPS化（TLS 1.2以上）
- パスワードはbcrypt等で ハッシュ化保存（平文保存は厳禁）
- 個人データへのアクセス制御（ロールベース）
- SQLインジェクション対策（プリペアドステートメント）
- XSS対策（出力エスケープ）
- CSRF対策
- 不正アクセスのログ記録
- 不要になったデータの確実な削除

推奨：
- DBへのアクセスはアプリ経由のみ（直接接続を禁止）
- 定期的な依存ライブラリの更新
- 個人データを含むファイルの暗号化（保存時）
```

### 外的環境の把握
外国で個人データを取り扱う場合（海外クラウド等）は、当該外国の個人情報保護に関する制度を把握した上で安全管理措置を講じること。

## 4. 漏えい等の報告

ガイドライン通則編 3-3-6（法第26条関係）より

### 「漏えい等」の考え方
- **漏えい**：個人データが外部に流出した状態
- **滅失**：個人データの内容が失われた状態
- **毀損**：個人データの内容が意図しない形で変更された状態、または変更はされていないが利用不能な状態

### 個人情報保護委員会への報告が義務となる事態（法第26条第1項）

以下のいずれかに該当する場合は**報告義務あり**：

| 事態 | 説明 |
|------|------|
| ①要配慮個人情報の漏えい等 | 病歴・障害・犯罪歴等を含む漏えい |
| ②財産的被害が生じるおそれ | クレジットカード番号・口座番号等の漏えい |
| ③不正の目的によると疑われる漏えい等 | 不正アクセス等の不正行為によるもの |
| ④1,000件を超える漏えい等 | 漏えいした個人データが1,000件超 |

### 報告期限
- **速報**：漏えい等を「知った」時点から**3〜5日以内**（おおむね）
- **確報**：速報から**30日以内**（不正目的によると疑われる場合は60日以内）

### 本人への通知（法第26条第2項）
報告義務が発生する事態が生じた場合、**本人への通知も義務**。  
通知が困難な場合は、本人が知りうる状態に置く措置（HPでの公表等）で代替可。

### 実際の漏えい原因の傾向（委員会の報告より）
- **不正アクセス**（ランサムウェア含む）：件数が多い
- **ヒューマンエラー**（紛失・誤送付・設定ミス）：件数が多い

不正アクセス事案の場合、委員会は**警察とIPA（情報処理推進機構）への連絡**も促している。

### 発覚後の対応フロー
```
1. 漏えい等の検知・認識
   ↓
2. 被害の拡大防止（アクセス遮断、パスワード変更等）
   ↓
3. 事実関係の調査（原因・規模・対象データの特定）
   ↓
4. 委員会への速報（3〜5日以内）
   ↓
5. 本人への通知（速やかに）
   ↓
6. 委員会への確報（30日以内）
   ↓
7. 再発防止策の実施・公表

※不正アクセス事案は警察・IPAへも連絡
報告フォーム：https://www.ppc.go.jp/personalinfo/legal/leakAction/
```

## 5. 第三者提供

ガイドライン通則編 3-3-7（法第27条〜第31条関係）より

### 原則：本人同意が必要（法第27条第1項）
個人データを第三者に提供する場合、**あらかじめ本人の同意**を得なければならない。

### 例外（本人同意不要）
- 法令に基づく場合（警察・裁判所からの照会等）
- 人の生命・身体・財産保護のため必要で、本人の同意取得が困難な場合
- 公衆衛生・児童の健全育成のため必要で、本人の同意取得が困難な場合
- 国の機関等への協力（税務調査、統計調査等）

### 「第三者」に該当しない場合（委託・事業承継・共同利用）
以下は「第三者提供」ではなく本人同意なしに情報共有可能：

| パターン | 条件 |
|---------|------|
| **委託**（法第27条第5項1号） | 委託先との間で適切な監督を行うこと |
| **事業承継**（同2号） | M&A等による事業譲受け |
| **共同利用**（同3号） | 共同利用する旨・利用目的・責任者等をあらかじめ通知または公表 |

→ **決済代行・配送業者等への情報提供は「委託」**として整理できる（本人同意不要だが委託先の監督義務あり）。

### オプトアウトによる第三者提供（法第27条第2項〜第4項）
一定の要件のもと、本人が拒否しない限り第三者提供できる仕組み。  
ただし：
- **個人情報保護委員会への届出**が必要
- **要配慮個人情報はオプトアウト不可**
- 不正取得した個人データはオプトアウト不可

### 個人関連情報の第三者提供制限（法第31条）
第三者が個人関連情報（Cookie等）を受け取り、それを既存の個人データと照合して個人識別する目的で使う場合、**提供元が本人同意を確認する義務**がある。

→ **広告配信・リターゲティング**でのCookie活用には注意が必要。

### 外国にある第三者への提供制限（法第28条）
外国の第三者（海外サービス等）に個人データを提供する場合：

1. **本人の同意**を得る（提供先の国名・個人情報保護制度の情報を提示の上）
2. または、**十分性認定国**への提供（EUの十分性認定取得済み）
3. または、提供先が**個人情報保護委員会規則で定める基準を満たす体制**を整備している場合

**開発実装での注意**：
```
以下を使う場合はプライバシーポリシーへの記載が必要：
- AWS（米国）、GCP、Azure等の海外リージョン
- Slack、Zoom、Salesforce等の米国SaaS
- SendGrid、Twilio等の海外メール・SMS配信サービス
- Stripe等の海外決済サービス
```

## 6. 保有個人データの開示・訂正・削除

ガイドライン通則編 3-3-8（法第32条〜第39条関係）より

### ユーザー（本人）の権利
| 請求の種類 | 内容 | 回答期限の目安 |
|-----------|------|--------------|
| 利用目的の通知請求（法第32条） | 保有する個人データの利用目的 | できるだけ速やかに |
| 開示請求（法第33条） | 保有個人データの内容 | 2週間以内 |
| 訂正・追加・削除請求（法第34条） | 内容が事実と異なる場合 | 2週間以内 |
| 利用停止・消去請求（法第35条） | 違法な取扱いがあった場合 | 2週間以内 |
| 第三者提供の停止請求（法第35条） | 違法な第三者提供があった場合 | 2週間以内 |
| 第三者提供記録の開示（法第33条第5項） | 誰に提供したかの記録 | 2週間以内 |

### 開示の方法（法第33条）
開示請求には、本人が求めた方法（電磁的記録の提供等）で対応すること。

### 開示等の手続き（法第37条）
- 本人確認のための合理的な手続きを定めることができる
- 手数料を設定可（実費の範囲内）

### 実装チェックリスト（ユーザー向け機能）
```
- [ ] アカウント削除機能（本人のデータを確実に消去）
- [ ] 自分のデータを確認・ダウンロードできる機能（開示対応）
- [ ] 登録情報の変更・訂正機能（訂正請求対応）
- [ ] 問い合わせ窓口の設置とプライバシーポリシーへの記載
- [ ] 開示請求等に対応する社内フローの整備
```

## 7. 開発実装チェックリスト

### フォーム・会員登録画面
```
- [ ] 利用目的をフォーム付近に明示（具体的な文言で）
- [ ] プライバシーポリシーへのリンクを設置
- [ ] 要配慮個人情報を取得する場合は同意チェックボックスを設置
- [ ] 取得必須項目と任意項目を明確に区別
- [ ] 未成年ユーザーを想定する場合は保護者同意の仕組みを検討
```

### データ管理・セキュリティ
```
- [ ] 通信のHTTPS化（TLS 1.2以上）
- [ ] パスワードのハッシュ化保存（bcrypt / argon2等）
- [ ] DBへのアクセス権限の最小化
- [ ] SQLインジェクション対策
- [ ] XSS・CSRF対策
- [ ] 個人データを含むログファイルの管理（マスキング等）
- [ ] 不要データの定期削除・廃棄
- [ ] バックアップデータの暗号化
```

### 外部サービス連携
```
- [ ] 解析ツール（GA4等）の利用をプライバシーポリシーに記載
- [ ] 広告配信（リターゲティング等）は同意バナーまたはオプトアウト手段を提供
- [ ] 海外SaaS・クラウド利用時は越境移転として記載
- [ ] 決済代行・配送業者等への提供は「委託」として整理し委託先監督
- [ ] Cookie利用の目的・種類を分類して記載（必須/分析/広告）
```

### プライバシーポリシー必須記載項目
```
1. 事業者の名称・住所・代表者名
2. 個人情報保護管理者（または問い合わせ窓口）
3. 取得する個人情報の種類
4. 利用目的（具体的に）
5. 第三者提供の有無・提供先・提供する情報・目的
6. 外国にある第三者への提供がある場合はその国名と制度概要
7. 委託先への情報提供（委託業務の内容）
8. 共同利用する場合はその旨・共同利用者・利用目的・責任者
9. 保有個人データの開示・訂正・削除等の手続き方法
10. Cookieその他のトラッキング技術の利用目的
11. お問い合わせ窓口（メールアドレス等）
12. プライバシーポリシーの改定手続き
```

### インシデント対応準備
```
- [ ] 不正アクセス検知の仕組みを用意（IDS、ログ監視）
- [ ] インシデント対応手順書の作成（誰が何をするか）
- [ ] 個人情報保護委員会への報告フォームをブックマーク
      https://www.ppc.go.jp/personalinfo/legal/leakAction/
- [ ] 警察・IPA（https://www.ipa.go.jp/）への連絡先を確認
- [ ] 本人通知の文面テンプレートを用意
```
